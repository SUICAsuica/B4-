## ドラフト書式の最優先遵守
- 現行ドラフト `draft_2025-11-26_v1.tex` の書式・体裁を最優先ルールとして採用する。以後の改稿はこのファイルを複製し、本文のみを更新する。

- 句読点は「，」「。」で統一し、段落先頭の全角1文字字下げを維持する。図表ラベルや見出しの付し方もドラフトと同一の位置・スタイルを踏襲する。

## 基本遵守事項
### 文章技術と文体
- 段落は1トピックに限定し、可能であればトピックセンテンスで始める。段落を付けない／1文ごとに段落を分けることは禁止。
- 読みにくい長文は分割し、必要に応じて文中箇条書きを用いて論点を整理する。
- 事実と意見を同一文に混在させない。事実は断定形、意見には「〜と考える／〜と推察する」などを付けて主観であることを明示する。
- 主語と述語の対応を常に確認し、省略する場合も暗黙の主語が誤解を生まないようにする。
- 文語体（「〜である」調）で統一し、口語・文学的比喩・体言止めを避ける。禁止表現と推奨表現の置換リストを参照し、必要に応じて修正する。
- 序論では「汎用知能基盤」「ビジョンサーボ」「タスク計画」など人工知能学会で一般的でない専門用語や意味圧縮した造語を使用しない。やむを得ず用いる場合は初出で平易な日本語による説明を付し、造語の新規作成は行わない。

### 章構成とストーリー
- 表紙 → 目次 → 序論 → 手法 → 結果 → 考察 → 結論 → 謝辞 → 参考文献 → 付録が基本構成。章名は研究に応じて具体化してよいが、序論は「背景→先行研究→目的→方法→論文構成」の順序を守る。
- 序論・結論はそれぞれ最低1ページ分の情報量を確保し、論旨が希薄にならないようにする。
- 手法章ではデータ取得方法・前処理・アルゴリズム・パラメータ・仮説を第三者が再現できる粒度で記述する。結果章では良否にかかわらず全実験を図表付きで報告し、本文で丁寧に解説する。

### レイアウトとフォーマット
- LaTeX テンプレート（`ltjsreport`、11pt、A4）を用い、余白設定は `top=20mm,bottom=30mm,left=20mm,right=20mm` を厳守する。本文フォントは LuaLaTeX-ja 既定の明朝系を使用し、サイズ・行間を変更しない。
- 表紙は `draft_2025-11-26_v1.tex` の表形式レイアウトを踏襲し、項目順・位置・フォントサイズ・下線長を変えない。表紙にページ番号を入れない。タイトルが複数行になる場合はドラフトと同等の自然な改行位置を用いる。
- 章区切り以外の任意改ページは禁止。LaTeX の自動組版に任せ、余白を意図的に残さない。ページ番号は目次を1ページ目とし中央下配置（デフォルト）を維持する。
- 目次は `\tableofcontents` を使用し、章立て変更後は再コンパイルで更新する。手動編集は禁止。

### 図表・参考文献・引用
- 図のタイトルは図の下、表のタイトルは表の上に置き、本文で必ず「図2に示す」「表3に示す」などと参照する。同じデータを図と表で重複提示しない。
- 図表内の文字・数値は判読可能な大きさに整える。スクリーンショットを使う場合は列揃えを確認し、必要に応じてExcel等で整形し直す。
- 他文献の図表を引用する場合はタイトルに「〇〇より抜粋」と明記する。本文では必ず出典を示す。
- 参考文献は本文で `[n]` 形式の参照を行い、学術論文は「著者，論文名，誌名，巻号，ページ，出版年」、書籍は「著者，書名，出版社，出版年」の順で統一する。ブログや卒論は原則不可。
- 他人の文章を丸写ししない。生成AIの出力を利用する際は一文ずつ真偽と文体を確認し、必要なら手動で修正する。

### 仕上げとチェック
- 段落の冒頭は全角1文字字下げ、句読点はドラフトに合わせ「，」「。」で統一する。
- 付録・図表・参考文献を本文で参照しているか、生成AIが挿入した誤情報がないかを最終確認する。PDF化後も文字化けやレイアウト崩れがないか確認する。
- 研究データ・原稿・発表スライドは更新のたびに複数媒体へバックアップを取る。
- 各研究テーマごとに、関連する論文や配布資料、実験データ、コード、分析結果を格納する専用リポジトリを新規に作成し、本リポジトリとは明確に分離して管理する。フォルダ構成やREADMEを整備し、研究メンバーが参照しやすい状態を保つ。
- 生成したドラフトや各章の文章はバージョン単位で必ず保存し、最新稿を読み込んでから追記・修正するサイクルを徹底する。保存は日付・版数付きのファイル名（例: `draft_2025-11-26_v1.tex`）とし、拡張子は `.tex` を用いる。以降、別命があるまでドラフトは Markdown ではなく TeX 形式で作成・更新する。

## 事前質問テンプレート（ユーザー確認）
論文執筆や修正に取りかかる前に、少なくとも以下の質問に対する回答を取得すること。回答が不足している場合は補足質問を行い、ログとして残す。
1. **研究テーマと目的**：対象・課題・期待する成果は何か。
2. **使用データと入手状況**：データソース、取得済みか、利用制限はあるか。
3. **手法・ツールの指定**：使用すべきアルゴリズムやモデル、計算環境、禁止手法の有無。
4. **評価指標・比較対象**：重視する評価基準やベンチマーク、比較対象モデル。
5. **図表・参考文献の要件**：必要な図表数、引用スタイル、最低文献件数やフォーマット指定。
6. **スケジュールとレビュー体制**：ドラフト提出期限、レビュー担当者、想定する版数管理。
7. **追加ガイドライン**：研究倫理、IRB、学内規定など他に従うべき資料や注意事項。

上記への回答が揃うまでは本文執筆を開始せず、得られた回答内容を逐次反映して執筆方針を更新すること。

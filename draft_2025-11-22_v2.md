# 1 序論
　近年，Vision-Language Model（以下 VLM）は画像理解と自然言語処理を統合した汎用知能基盤として注目され，移動ロボットのナビゲーションやビジョンサーボへの応用が進んでいる。SayCan [1] や PaLM-E [2] は大規模言語モデルを用いたタスク計画を実現し，RT-2 [3] は画像と言語から直接行動を生成する Vision-Language-Action を示した。また VL-Maps [4] はセマンティック地図を事前に構築し，探索性能を高めている。これらは地図，外部メモリ，階層計画といった補助機構を備え，高性能だが計算資源や準備コストが大きい。一方，低演算資源の小型移動体において，VLM を「視覚→行動」変換器として素朴に用いた場合の基礎的性能と限界は十分に検証されていない。
　本研究は，OSOYOO Raspberry Pi Car を対象に，地図や複雑なセンサフュージョンを用いない「Mapless & Minimal-Sensor」構成で，VLM 単体制御がどこまで安定走行を実現できるかを実験的に評価する。タスクは「黄色の TARGET 箱へ接近し，前方 20 cm 以内で停止するビジョンサーボ」である。システムはラズパイ上の軽量エージェント（`raspi_agent`）が映像転送とモータ駆動を担い，推論は PC にオフロードする。ロボット側に地図生成や SLAM を搭載せず，VLM の視覚判断に依存する点で低コスト性と実装の単純さを重視する。
　使用モデルは GPT-5-mini-20250807 を主とし，軽量モデル（SmolVLM 系）の試行を付録で報告してモデル選択の妥当性を示す。評価指標は到達時間，成功率，コマンド適合率を主軸とし，補助指標としてコマンド反転回数と推論遅延分布を計測する。比較条件として，(A) VLM が直接コマンドを出力するダイレクト制御と，(B) VLM は位置・距離の認識のみを返し，決定論的ルールでコマンドを生成するハイブリッド制御を，同一速度・ループ間隔で実施する。
　本研究の貢献は三点である。第一に，安価な移動体と VLM の最小構成における実世界走行性能を定量化し，基礎的限界を明示する。第二に，ダイレクト制御とハイブリッド制御の比較を通じ，VLM が判断と制御のどちらで誤りを生じやすいかを分離して示す。第三に，失敗事例を分析し，外部メモリ導入や世界モデル統合に向けた設計要件を提示することで，将来的なエンドツーエンド自律走行研究の足場を提供する。

# 2 手法
## 2.1 ハードウェア構成
　対象機体は OSOYOO Raspberry Pi Robot Car（2023版）であり，Raspberry Pi 4B，広角 USB カメラ，モータドライバ（L298N），ステアリング用サーボから成る。カメラ解像度は 640×480，フレームレートは 5 fps とし，室内平坦路面で走行させる。

## 2.2 通信とソフトウェア構成
　ラズパイ上で `raspi_agent.py` を 8080 番ポートで起動し，`GET /frame.jpg` および `/stream.mjpg` によるフレーム配信と，`POST /command` による `FORWARD/LEFT/RIGHT/BACKWARD/STOP` コマンド受信を提供する。PC 側は同一ネットワークから HTTP でアクセスし，VLM 推論結果に基づいてコマンドを送信する。環境切替のため，自宅・研究室それぞれの IP を環境変数で管理し，実験時の誤接続を防ぐ。

## 2.3 使用モデルとプロンプト
　主モデルは GPT-5-mini-20250807（ローカル実行）とし，単一フレームから最適コマンドを一語で出力させる。プロンプトは「黄色の TARGET 箱に到達して停止せよ」を明示し，選択肢を `LEFT/RIGHT/FORWARD/FORWARD_SLOW/STOP/BACKWARD` に限定する。SmolVLM 系モデルも試行したが，到達率の低下と誤指示が顕著であり，付録に失敗要因を記述する。

## 2.4 制御条件
- 条件A（ダイレクト）: VLM が各ループでコマンドを直接選択し，そのまま送信する。ループ間隔は 0.5 秒，速度は固定とする。
- 条件B（ハイブリッド）: VLM は「ターゲット位置（LEFT/CENTER/RIGHT）と距離（FAR/MID/NEAR）」を JSON 形式で返し，PC 側で単純なルールによりコマンドを決定する。ルールは「CENTER なら前進，NEAR なら減速または停止，LEFT/RIGHT なら旋回」とする。
　両条件ともヒステリシスは用いず，VLM 出力と単純ルールのみで挙動を決める。

# 3 実験
## 3.1 環境とコース
　室内直線コース（長さ 2.5 m）と L 字コース（1.5 m＋1.5 m）を作成し，照度は 400–500 lx に保つ。壁や障害物は固定し，背景テクスチャを一定にする。ターゲットは黄色の箱に “TARGET” と印字したものを用い，終点に配置する。

## 3.2 手順と試行数
　各条件について各コース10試行ずつ実施し，計40試行を行う。走行中はフレーム，VLM 出力，送信コマンド，タイムスタンプを保存し，別途カメラで全体動画を記録する。試行ごとに ID を付与し，メタ情報（コース種別，照度，モデルバージョン，開始・終了時刻）を記録する。

## 3.3 評価指標
- 到達時間：開始コマンド送信から停止判定までの時間。
- 成功率：ターゲット前 20 cm 以内で停止した試行の割合。
- コマンド適合率：全送信コマンド中，理想方向・速度と評価できる割合。
- コマンド反転回数：左右または前後の即時反転回数。
- 推論遅延：VLM 応答時間の中央値と 95 パーセンタイル。

# 5 考察（予定）
　予備結果では，ダイレクト制御は推論遅延と誤指示によりコマンド反転が増え，到達時間がばらつく傾向が見られると予想される。ハイブリッド制御は距離推定の粗さが残るものの，左右誤指示の影響が緩和され成功率が高まると見込む。VLM 単体制御の主なボトルネックは (1) 応答遅延がループ間隔に近接すること，(2) 単一フレーム依存による一時的誤認識，(3) 照度変動への感度である。これらを踏まえ，外部メモリやフレームスタックによる時系列文脈の導入，軽量世界モデルによる予測補正が有効と考える。

# 6 結論
　本研究は，低コスト移動体において VLM 単体制御が達成し得る性能を明らかにすることを目的とし，ダイレクト制御とハイブリッド制御を同条件で比較する実験計画を示した。主指標として到達時間・成功率・コマンド適合率を設定し，補助指標として反転回数と遅延分布を測定する枠組みを構築した。今後，実験データを反映した結果と考察を追記し，外部メモリ統合や予測モデルを含む拡張方針を具体化する。

# 付録A SmolVLM 失敗試行の概要
　SmolVLM2-mlx を用いて同一タスクを試行したところ，推論時間が平均 0.8 s と長く，ループ間隔 0.5 s を上回った。結果として過去フレームに基づく誤指示が頻発し，10 試行中の成功は 2 回に留まった。誤指示の内訳は LEFT/RIGHT の即時反転が 12 件，停止不能が 4 件であった。今後は軽量化またはフレーム間補間を検討する。
